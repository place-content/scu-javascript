# TaskFlow - 개인 업무 관리 플랫폼 비즈니스 모델 기획서

## 1. 사업 개요

### 1.1 서비스명
**TaskFlow** - 간단한 개인 업무 관리 도구

### 1.2 사업 목표
개인 사용자들의 일상 업무와 할 일을 체계적으로 관리할 수 있는 간단하고 직관적인 웹 애플리케이션 제공

### 1.3 시장 문제점
- 복잡한 프로젝트 관리 도구는 개인 사용자에게 과하게 복잡함
- 종이나 메모장으로 관리하는 할 일 목록의 비효율성
- 마감일 잊어버림으로 인한 업무 누락 문제
- 여러 기기 간 동기화의 어려움

## 2. 비즈니스 모델

### 2.1 수익 모델
- **프리미엄 구독** (월 2,900원):
  - 무제한 할 일 생성
  - 카테고리별 분류 기능
  - 파일 첨부 기능
  - 이메일 리마인더
- **광고 수익**: 무료 사용자에게 표시되는 배너 광고
- **B2B 패키지** (기업용, 월 29,000원): 팀원 초대, 공유 작업 공간

### 2.2 타겟 고객
- **1차 타겟**: 대학생, 직장인 (개인 생산성 향상)
- **2차 타겟**: 프리랜서, 자영업자
- **3차 타겟**: 소규모 팀 및 스타트업

## 3. 서비스 특장점

### 3.1 간편함과 직관성
- 3초 만에 할 일 추가 가능
- 드래그앤드롭으로 순서 변경
- 실시간 자동 저장

### 3.2 스마트한 관리
- 마감일 알림 기능
- 완료된 항목 자동 아카이빙
- 통계 및 진행률 시각화

### 3.3 접근성
- 모든 기기에서 접근 가능 (반응형 웹)
- 오프라인 기능 지원
- 다국어 지원 (한국어, 영어)

## 4. 소프트웨어 아키텍처

### 4.1 프론트엔드 (Frontend)
- **기술 스택**: HTML5, CSS3, JavaScript (Vanilla JS), Bootstrap
- **주요 기능**:
  - 반응형 웹 디자인
  - SPA (Single Page Application) 구현
  - 로컬 스토리지 활용 오프라인 기능
  - RESTful API 통신

### 4.2 백엔드 (Backend)
- **기술 스택**: Node.js, Express.js, JavaScript
- **주요 기능**:
  - RESTful API 제공
  - JWT 기반 인증
  - 파일 업로드 처리
  - CORS 지원

### 4.3 데이터베이스 (Database)
- **주요 DB**: MongoDB (NoSQL)
- **데이터 모델**:
  - 사용자 정보 (Users)
  - 할 일 목록 (Tasks)
  - 카테고리 (Categories)
  - 완료 기록 (History)

### 4.4 웹 서버 및 인프라
- **웹 서버**: Nginx (정적 파일 서빙)
- **배포**: Docker 컨테이너
- **클라우드**: AWS 또는 Heroku
- **CDN**: CloudFlare (글로벌 콘텐츠 전송)

## 5. 핵심 기능 (CRUD)

### 5.1 생성 (Create)
- 새 할 일 추가
- 카테고리 생성
- 마감일 설정
- 우선순위 지정

### 5.2 읽기 (Read)
- 할 일 목록 조회
- 상세 정보 보기
- 필터링 및 검색
- 정렬 기능

### 5.3 업데이트 (Update)
- 할 일 내용 수정
- 완료 상태 변경
- 마감일 조정
- 카테고리 이동

### 5.4 삭제 (Delete)
- 할 일 삭제
- 카테고리 삭제
- 완료된 항목 정리

## 7. 개발 스택 상세

### 7.1 프론트엔드 구조
```
public/
├── index.html          # 메인 HTML 파일
├── css/
│   └── style.css       # 스타일시트
├── js/
│   ├── app.js          # 메인 애플리케이션 로직
│   ├── auth.js         # 인증 관련 기능
│   ├── task.js         # 할 일 관리 기능
│   └── utils.js        # 유틸리티 함수
└── assets/             # 이미지 및 기타 자산
```

### 7.2 백엔드 구조
```
server/
├── app.js              # Express 서버 메인 파일
├── config/
│   └── database.js     # 데이터베이스 설정
├── models/
│   ├── User.js         # 사용자 모델
│   └── Task.js         # 할 일 모델
├── routes/
│   ├── auth.js         # 인증 라우트
│   └── tasks.js        # 할 일 관리 라우트
└── middleware/
    └── auth.js         # 인증 미들웨어
```

## 8. 데이터베이스 설계

### 8.1 사용자 컬렉션 (Users)
```javascript
{
  _id: ObjectId,
  email: String,
  password: String,
  name: String,
  createdAt: Date,
  updatedAt: Date,
  subscription: String  // 'free' | 'premium'
}
```

### 8.2 할 일 컬렉션 (Tasks)
```javascript
{
  _id: ObjectId,
  userId: ObjectId,     // 사용자 ID (외래키)
  title: String,        // 할 일 제목
  description: String,  // 상세 설명
  category: String,     // 카테고리
  priority: Number,     // 우선순위 (1-5)
  dueDate: Date,        // 마감일
  completed: Boolean,   // 완료 여부
  createdAt: Date,
  updatedAt: Date
}
```

## 9. API 설계

### 9.1 인증 API
- POST `/api/auth/register` - 회원가입
- POST `/api/auth/login` - 로그인
- POST `/api/auth/logout` - 로그아웃

### 9.2 할 일 관리 API
- GET `/api/tasks` - 할 일 목록 조회
- POST `/api/tasks` - 할 일 생성
- PUT `/api/tasks/:id` - 할 일 수정
- DELETE `/api/tasks/:id` - 할 일 삭제
